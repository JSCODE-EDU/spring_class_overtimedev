이제 ‘기록하는 습관’을 들여볼겁니다. 작성해야 되는 내용은 아래와 같습니다.

- 배운 내용, 깨달은 점
    git pull, git branch를 따로 만들어서 사용할 수 있는 것을 알게 되었다.
- 어려웠던 점, 반성하고 싶은 점 / 개선할 방법
    git에 대한 이해도가 너무 부족해서 그런지 origin이 무엇인지 branch가 무엇인지, pull, push, fetch가 무엇인지 몰랐다.
    또한 이미지화가 잘 안되서 어렵기도 했다. 이를 보완하기 위해 git에 대해서 공부를 해야하는데 
    단순히 책만보고 따라만 하면 금방 또 까먹을 것 같다.
    오늘과 같이 1주차 미션을 따라하면서 배우니 습득 속도도 훨씬 빨랐던 것 같다.
    또한 누군가에게 물어볼 수 있는 것 또한 축복이다. 원래라면 혼자서 고독하게 공부해야 했을 것이다.
- 미션 제출
    - 1주차 미션 clone push 후 PR요청하기 (PR은 못했지만 clone 후 push함)
- 궁금한 점
    - 아직 spring에 대해서 질문할 때는 아닌 것 같아 git에 대해 질문하고 싶습니다.
      1. 브랜치를 main위에는 어떻게 만드나요?
          이제까지 해온 방법---> git branch overtimedev
                                        git branch 
                                        
                                        master. overtimedev 두 개 존재 --> 이 둘은 서로 완전 다른 branch임
      2.만약 두 branch가 서로 다르다면 branch의 내용도 완전 다른가요?
        그렇다면 checkout으로 바꿔줄 때마다 파일은 변하나요?
      3. PR은 왜 하는 것인가요? 아직 용도를 잘 모르겠습니다.
      4. git에 나온 모든 부분을 당장 학습해야하나요? -->자꾸 다 까먹습니다.
      5.대학이 꼭 필요할까요? --> 좋지 않은 학력이기도 하고 대학을 잘 다닌 것인지 겁이 납니다.
